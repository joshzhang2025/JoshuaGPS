# Joshua GPS 部署流程与端口指南

> 本文档面向部署人员，完整覆盖从环境检查 → 部署 → 启动 → 防火墙放行 → 验证的全流程。

---

## 一、项目性质（先搞清楚再动手）

本项目是**纯静态网站**，不需要：

- ❌ 后端服务 / 数据库
- ❌ npm install / Node 构建
- ❌ 登录系统

只需要：

- ✅ 一个能托管静态文件的 Web 服务器
- ✅ 正确返回 `.html`、`.css`、`.js`、`.json` 的 MIME 类型
- ✅ 服务必须在**项目根目录**启动

---

## 二、部署方式对比

| 方式 | 适用场景 | 稳定性 | 推荐度 |

## 三、部署全流程（Python 临时服务）

### 第 1 步：确认文件完整性

项目根目录必须包含以下文件结构：

```
项目目录/
├── index.html              ← 主页面入口
├── deployment-guide.html   ← 部署说明页
├── style.css               ← 样式文件
├── script.js               ← 主脚本（通过 fetch 加载 JSON）
├── data/
│   ├── gps.json            ← 主数据源（核心文件）
│   ├── today.json
│   ├── daily-logs.json
│   └── project-notes.json
└── server-config/
    ├── nginx-joshua-gps.conf
    └── web.config
```

> ⚠️ 缺少 `data/gps.json` 时首页不会显示任何内容。

### 第 2 步：确认 Python 版本

```powershell
python --version
```

要求：**3.10 或以上**（优先 3.11 / 3.12）。

如果安装了多个版本，尝试：

```powershell
py -3 --version
```

### 第 3 步：确认当前目录

```powershell
cd /d 项目完整路径
```

⚠️ **关键**：必须切换到项目根目录（即包含 `index.html` 的目录），否则服务找不到文件。

### 第 4 步：选择端口

```powershell
# 查看 10000-10010 范围内哪些端口未被占用
10000..10010 | ForEach-Object { $p = $_; $found = netstat -ano | Select-String ":$p "; if (!$found) { [PSCustomObject]@{ Port = $p; Status = "可用" } } }
```

推荐使用 **10001** 端口（本项目统一约定）。如果被占用，顺延到下一个可用端口。

### 第 5 步：启动服务

```powershell
python -m http.server 10001
```

**正常状态**：终端显示以下信息后**保持窗口开着**，不要关闭：

```
Serving HTTP on :: port 10001 (http://[::]:10001/) ...
```

这不是卡住了，这是**服务正在运行**。终端会实时打印访问日志：

```
127.0.0.1 - - [15/Jun/2026 ...] "GET / HTTP/1.1" 200 -
127.0.0.1 - - [15/Jun/2026 ...] "GET /script.js HTTP/1.1" 200 -
```

### 第 6 步：防火墙放行端口

> 如果只有本机能访问、其他设备连不上，通常是防火墙阻止了入站连接。

#### 6.1 检查端口是否已被放行

```powershell
$port = 10001
$rule = Get-NetFirewallRule -Direction Inbound -Action Allow -Enabled True |
  Get-NetFirewallPortFilter |
  Where-Object { $_.Protocol -eq "TCP" -and $_.LocalPort -eq "$port" }
if ($rule) { "端口 $port 已放行" } else { "端口 $port 未放行" }
```

#### 6.2 以管理员身份打开 PowerShell

方式一：按 Win → 输入 `PowerShell` → 右键 → "以管理员身份运行"

方式二：在已有终端中执行：

```powershell
Start-Process powershell -Verb RunAs
```

#### 6.3 添加防火墙放行规则

```powershell
New-NetFirewallRule -DisplayName "Joshua GPS 10001" -Direction Inbound -Protocol TCP -LocalPort 10001 -Action Allow -Profile Any
```

参数说明：

| 参数 | 含义 |
|---|---|
| `-DisplayName` | 规则名称（自定义，方便识别） |
| `-Direction Inbound` | 入站规则（别人连进来） |
| `-Protocol TCP` | HTTP 使用 TCP 协议 |
| `-LocalPort 10001` | 要放行的端口号 |
| `-Action Allow` | 允许通过 |
| `-Profile Any` | 对所有网络类型生效（域/专用/公用） |

#### 6.4 再次验证

```powershell
$port = 10001
$rule = Get-NetFirewallRule -Direction Inbound -Action Allow -Enabled True |
  Get-NetFirewallPortFilter |
  Where-Object { $_.Protocol -eq "TCP" -and $_.LocalPort -eq "$port" }
if ($rule) { "端口 $port 已放行" } else { "端口 $port 未放行" }
```

显示 **"已放行"** 即可。

#### 6.5 如需删除规则

```powershell
Remove-NetFirewallRule -DisplayName "Joshua GPS 10001"
```

### 第 7 步：访问验证

| 访问位置 | 地址 |
|---|---|
| 本机浏览器 | http://127.0.0.1:10001/ |
| 局域网其它设备 | http://服务器IP:10001/ |
| 数据文件直连 | http://服务器IP:10001/data/gps.json |
| 部署说明页 | http://服务器IP:10001/deployment-guide.html |

**查服务器 IP**：

```powershell
ipconfig | findstr IPv4
```

### 第 8 步：验收清单

- [ ] 首页样式加载正常，不是纯文字
- [ ] 左侧导航可切换各模块
- [ ] `http://服务器IP:10001/data/gps.json` 能直接返回 JSON
- [ ] 中英文切换按钮可用
- [ ] Read current page 可触发朗读
- [ ] Daily Log 表单可以提交
- [ ] 手机端无横向滚动

## 四、端口详解

### 4.1 端口基础知识

| 范围 | 类型 | 说明 |
|---|---|---|
| 0 - 1023 | 知名端口 | 系统服务专用，需管理员权限 |
| 1024 - 49151 | 注册端口 | 应用程序常用 |
| 49152 - 65535 | 动态/私有端口 | 临时分配 |

本项目使用的 **10001** 属于注册端口范围，普通用户权限即可监听。

### 4.2 常用端口操作命令

```powershell
# 查看指定端口是否被占用
netstat -ano | findstr :10001

# 查看所有正在监听的端口
netstat -ano | findstr LISTENING

# 查看指定端口的进程名称
netstat -ano | findstr :10001
tasklist | findstr PID号

# 强制结束占用端口的进程
taskkill /PID 进程号 /F
```

### 4.3 端口问题排查流程

```
页面无法访问
    │
    ├─ 服务是否启动？
    │     └─ netstat -ano | findstr :端口号
    │         ├─ 无 → 执行 python -m http.server 端口号
    │         └─ 有 → 下一步
    │
    ├─ 端口是否正确？
    │     └─ 确认访问地址端口与服务启动端口一致
    │
    ├─ 防火墙是否放行？
    │     └─ 用 6.1 的命令检查 → 未放行则执行 6.3
    │
    └─ 是否从项目根目录启动？
          └─ 检查 index.html 是否在启动目录下
```

### 4.4 端口被占用的解决方案

方案一：**换端口**（推荐，简单快捷）

```powershell
python -m http.server 10002   # 换一个端口
```

方案二：**杀掉占用进程**

```powershell
netstat -ano | findstr :10001   # 找到 PID
taskkill /PID 进程号 /F           # 强制结束
```

## 五、IIS 正式部署流程

### 第 1 步：安装 IIS

```powershell
# 在 Windows Server 上安装 IIS（管理员 PowerShell）
Install-WindowsFeature -Name Web-Server -IncludeManagementTools
```

### 第 2 步：准备部署目录

```powershell
mkdir E:\site\Joshua GPS
# 将项目所有文件复制到该目录
```

### 第 3 步：部署 web.config

将 `server-config\web.config` 复制到 `E:\site\Joshua GPS\` 根目录。该配置确保：

- `.json` → `application/json`（关键！）
- `.js` → `application/javascript`
- `.css` → `text/css`
- 默认文档包含 `index.html`
- 禁用缓存（方便调试）

### 第 4 步：IIS 管理器中添加站点

1. 打开 IIS 管理器
2. 右键 **Sites** → **Add Website**
3. 站点名称：`Joshua GPS`
4. 物理路径：`E:\site\Joshua GPS`
5. 绑定类型：`http`，IP 地址：`192.168.0.224`，端口：`10001`
6. 确定后启动站点

### 第 5 步：防火墙放行（同第三部分第 6 步）

### 第 6 步：访问验证

```
http://192.168.0.224:10001/index.html
http://192.168.0.224:10001/data/gps.json
```

## 六、Nginx 部署流程（Linux）

### 第 1 步：安装 Nginx

```bash
sudo apt install nginx    # Ubuntu/Debian
sudo yum install nginx    # CentOS/RHEL
```

### 第 2 步：准备部署目录

```bash
sudo mkdir -p /var/www/joshua-gps
# 将项目文件上传到该目录
```

### 第 3 步：配置 Nginx

复制 `server-config/nginx-joshua-gps.conf` 并修改：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;          # ← 必须修改

    root /var/www/joshua-gps;           # ← 确认路径正确
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 第 4 步：启用配置并重启

```bash
sudo ln -s /etc/nginx/sites-available/joshua-gps /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 第 5 步：防火墙放行（Linux）

```bash
# 使用 firewalld
sudo firewall-cmd --add-port=80/tcp --permanent
sudo firewall-cmd --reload

# 或使用 iptables
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
```

### 第 6 步：访问验证

```
http://你的域名或IP/index.html
```

## 七、常见问题速查

| 问题 | 原因 | 解决 |
|---|---|---|
| 页面空白无内容 | `data/gps.json` 未正确加载 | 检查 `http://IP:端口/data/gps.json` 能否访问 |
| 页面乱码或部分文字显示异常 | 文件编码问题 | 确认所有文件为 UTF-8 编码 |
| 只有本机能访问 | 防火墙未放行 | 执行防火墙放行命令 |
| 端口冲突报错 | 端口已被占用 | 换端口或杀掉占用进程 |
| 服务启动后终端无输出 | 正常，等浏览器访问才会有日志 | 不要关闭终端，直接打开浏览器访问 |
| 双击 index.html 打开后页面空白 | 浏览器的 CORS 安全策略阻止了 fetch | 必须通过 HTTP 服务访问，不要用 file:// |
| 手机端/其他设备访问卡慢 | 网络问题或防火墙 | 确认在同一局域网，防火墙已放行 |
| IIS 返回 JSON 时下载而不是显示 | MIME 类型未配置 | 检查 web.config 是否已部署到根目录 |
